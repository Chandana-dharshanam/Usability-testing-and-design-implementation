Key findings
CriticalPromo code field not discoverable
4 of 5 participants failed to find the coupon input. It sits below the fold with no visual cue — only 40% task success rate.
CriticalPayment screen lacks trust signals
3 participants hesitated or abandoned at card entry. No padlock, no card brand logos, no "secure checkout" copy visible.
ModerateBack navigation breaks cart context
2 participants lost cart state pressing the device back button — expected to return to cart, got the product listing instead.
MinorOrder tracking mental model mismatch
Users expected email confirmation first — the in-app "My orders" label is not prominent enough to be found naturally.
Design recommendations
Elevate promo code input P1
Move coupon field above the order summary fold. Add a "Have a promo code?" collapsible link with clear CTA.
Add trust indicators to payment screen P1
Add SSL badge, Visa/Mastercard logos, and "256-bit encrypted" copy near the card input field.
Fix back-navigation state retention P2
Persist cart in session so back navigation from checkout returns to cart — not the product list.
Improve post-purchase confirmation P3
Show a prominent success screen with order number and "Track order" CTA immediately after purchase.
